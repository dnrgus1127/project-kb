# 흐름도·SVG 디자인 가이드 (diagram-design)

> `conventions.md` §7이 가리키는 **시각 품질의 상세 출처**다.
> §7은 "어디에 저장하고 어떻게 임베드하나"(파일·경로·viewBox·무의존)를 규정하고,
> 이 파일은 "어떻게 그려야 읽히나"(색·패키지별 색 구분·모양·픽토그램·라이프사이클 장치·가독성·테마)를 규정한다.
> 흐름도/구성도/시퀀스도/라이프사이클도 SVG를 만들거나 갱신하는 모든 스킬(`kb-update`, `kb-status`)이 따른다.

---

## 1. 철학

- **글보다 그림**: 흐름·구조·분기는 문장으로 풀지 말고 도식으로 보여준다.
- **색과 모양에 의미를 담는다**: 장식이 아니라 정보다. 같은 의미는 항상 같은 색·모양.
- **로컬·무의존**: 외부 폰트·아이콘폰트·스크립트·래스터 이미지에 의존하지 않는다. 보관함은 오프라인 로컬 파일이다.
- **테마 안전**: Obsidian을 다크 테마로 봐도 라이트로 봐도 동일하게 읽혀야 한다(§6).

---

## 2. 색상 시스템 (의미 색 토큰)

카테고리마다 `fill`(채움) · `stroke`(테두리) · `text`(글자) **3종 세트**를 정해 두고 그대로 쓴다.
모두 밝은 배경 패널(§6) 위에서 라이트·다크 양쪽 모두 충분한 대비를 갖도록 고른 값이다.

| 의미 | 예시 노드 | `fill` | `stroke` | `text` |
|---|---|---|---|---|
| 입력·소스 | 데이터 소스, 외부 피드, 사용자 입력 | `#DBEAFE` | `#2563EB` | `#1E3A8A` |
| 처리·엔진 | 변환, 계산, 비즈니스 로직 | `#F1F5F9` | `#64748B` | `#1E293B` |
| 판단·분기 | 조건, 게이트, 라우팅 | `#FEF3C7` | `#D97706` | `#78350F` |
| 데이터 저장소 | DB, 캐시, 파일, 큐 | `#EDE9FE` | `#7C3AED` | `#4C1D95` |
| 출력·정상종료 | 결과, 알림, 정상 끝 | `#DCFCE7` | `#16A34A` | `#14532D` |
| 위험·경보·오류종료 | 실패, 예외, 경보, 오류 끝 | `#FEE2E2` | `#DC2626` | `#7F1D1D` |
| 외부 시스템·액터 | 제3자 API, 외부 서비스, 사람 | `#F8FAFC` | `#94A3B8` (dashed) | `#334155` |

보조 색:
- **화살표·연결선**: `#475569` (선 굵기 1.5~2px). 화살표 라벨 글자도 동일 계열.
- **강조 화살표**(주 경로): 굵기 2.5px 또는 stroke를 해당 흐름의 색 토큰으로.

> [!warning] 색만으로 의미를 전달하지 않는다
> 색약 사용자와 흑백 인쇄를 고려한다. 의미 구분은 **색 + 모양 + 라벨**을 함께 쓴다.
> 예: 위험은 빨강만이 아니라 빨강 + ⚠ 픽토그램 + "오류" 라벨.

## 3. 노드 종류별 시각 표준 (종단점 포함)

같은 역할의 노드는 항상 같은 모양으로 그린다. **시작·끝 터미널은 굵은 테두리로 강조**해 흐름의 진입/종료를 한눈에 보이게 한다.

| 종류 | 모양 | 색 토큰 | 픽토그램(권장) | 비고 |
|---|---|---|---|---|
| **시작 터미널** | stadium(둥근 알약) | 출력·초록 | ▶ 또는 ● | 테두리 굵게(2.5px). 흐름의 단일 진입점. |
| **정상 끝 터미널** | stadium | 처리·짙은 회색 | ■ | 정상 종료. 테두리 굵게. |
| **오류 끝 터미널** | stadium | 위험·빨강 | ⚠ | 비정상 종료. 정상 끝과 색으로 구분. |
| 처리 | 직사각형(둥근 모서리 4px) | 처리·회색 | — | 가장 흔한 기본 노드. |
| 판단·분기 | 마름모 | 판단·앰버 | ? | 나가는 화살표에 Yes/No·조건 라벨. |
| 데이터 저장소 | 원통(cylinder) | 저장소·보라 | 🗄 실루엣 | DB/캐시/파일/큐. |
| 외부 시스템 | 점선 직사각형 | 외부·점선 회색 | ☁/⬡ | 우리가 통제 못 하는 경계 밖. |
| 액터(사람·역할) | 점선 사각형 또는 사람 픽토그램 | 외부·점선 회색 | 👤 실루엣 | 사용자·운영자 등. |
| 경보·위험 처리 | 직사각형 + 빨강 테두리 | 위험·빨강 | ⚠ | 경보 발생·차단 등. |

> [!tip] 시작/끝을 반드시 구분
> 모든 흐름도는 시작 터미널 1개에서 출발하고, 정상 끝 / 오류 끝으로 닫힌다.
> 끝이 정상인지 오류인지 색으로 즉시 구분되게 한다(회색 vs 빨강).

## 4. 아이콘·픽토그램 규칙

- **인라인으로만 그린다**: SVG 기본 도형(`<rect>`, `<circle>`, `<path>`)으로 직접 그린다. 외부 이미지 파일·아이콘 폰트·이모지 글꼴 의존 금지(이모지는 표 안의 설명용 표기일 뿐, 렌더 의존 대상이 아니다).
- **단순 실루엣**: 의미만 전달하는 최소 형태. 그라데이션·그림자·과한 디테일 금지.
- **라벨과 병행**: 픽토그램은 텍스트 라벨을 보조한다. 픽토그램만으로 의미를 떠넘기지 않는다.
- **크기·위치 일관**: 같은 다이어그램 안에서 픽토그램 크기(예: 16~20px)와 배치(노드 좌상단 또는 라벨 앞)를 통일한다.
- **색**: 픽토그램은 해당 노드의 `stroke` 색을 따른다(별도 색 도입 금지).

## 5. 가독성 규칙

- **폰트**: `font-family="-apple-system, 'Apple SD Gothic Neo', sans-serif"`. 한글은 그대로 둔다.
- **최소 글자 크기**: 본문 라벨 13~14px 이상, 제목 16px 이상. 이보다 작게 쓰지 않는다.
- **라벨 줄바꿈**: 긴 라벨은 `<tspan>`으로 2줄까지 나눈다. 노드 밖으로 넘치지 않게.
- **노드 최소 크기·여백**: 텍스트 상하좌우에 최소 8px 패딩. 노드끼리 최소 24px 간격.
- **흐름 방향 일관**: 한 다이어그램은 LR(가로) 또는 TB(세로) 한 방향으로. 역류 화살표 최소화, 선 교차 최소화.
- **선·화살표**: 굵기 1.5~2px, 화살촉 크기 일관. 화살표에는 분기 조건 등 필요한 라벨만.
- **대비**: `text`는 자기 노드 `fill` 위에서 충분히 읽혀야 한다(§2 토큰은 이를 보장한다). 임의 색을 쓸 땐 명도 대비를 직접 확인한다.
- **캔버스 여백**: 콘텐츠 가장자리에 16~24px 여백을 둔다.

## 6. 테마 안전 (다크 모드)

> [!warning] 투명 배경 + 검은 글자 = 다크 테마에서 안 보임
> 배경을 비워 두면 Obsidian 다크 테마에서 어두운 캔버스에 어두운 글자가 겹쳐 사라진다.

규칙:
- **명시적 배경 rect**: SVG 맨 아래(z-순서상 맨 뒤)에 캔버스 전체를 덮는 밝은 패널을 그린다.
  `<rect width="100%" height="100%" fill="#FFFFFF"/>` 또는 `#FCFCFD` + 옅은 테두리 `#E2E8F0`.
- 이 밝은 패널 위에서 §2 토큰을 쓰면, Obsidian 테마와 무관하게 항상 동일하게 보인다.
- `currentColor`·테마 의존 색을 쓰지 않는다. 모든 색을 명시 값으로 박는다.
- 작성 후 **라이트·다크 두 테마 모두에서** 가독성을 점검한다.

## 7. 흐름도 패턴 (무엇을 어떻게 그리나)

흐름도는 한 종류가 아니다. 문서가 설명하려는 것에 맞춰 패턴을 고른다.

| 패턴 | 언제 | 축·레이아웃 | 색의 의미 |
|---|---|---|---|
| 처리 흐름도 | 입력→처리→출력, 의사결정 분기 | LR 또는 TB, 단일 진입→종단 | 흐름 역할(§2: 입력/처리/판단/저장/출력/오류) |
| 시스템 구성도 | 모듈·계층·배치 관계(정적) | 자유 배치 | **패키지·계층(§7.1)** |
| 시퀀스·라이프사이클도 | "언제(호출 순서) 무엇이 일어나나", 협업/왕복 | **세로 시간축(위=먼저, 아래=나중)** | **패키지·계층(§7.1)** + 경로 레인 |

### 7.1 패키지별 색 구분 (구성도·라이프사이클의 기본 색 체계)

§2의 색 토큰은 **흐름 역할**(입력/처리/판단…)에 색을 준다. 그러나 **여러 패키지·모듈·계층이 한 흐름에 얽힌 문서**(예: 공유 엔진 ↔ 호스트 앱 ↔ 프레임워크)에서는 색을 **"누구의 코드인가(패키지·계층·소유권)"** 에 배정해 **패키지별로 구분**하는 편이 한눈에 읽힌다. 이런 흐름도는 패키지 색 구분을 **기본**으로 한다.

- **패키지마다 `fill`·`stroke`·`text` 한 세트**를 정하고 끝까지 그대로 쓴다. **같은 패키지 노드는 항상 같은 색.**
- **범례에 "색 = 패키지" 매핑을 반드시 명시**한다(예: A모듈=파랑, 공유 엔진=주황, 호스트=분홍, 프레임워크=회색, 중립/터미널=흰색+굵은 테두리).
- **화살표도 경로(레인)별로 색을 맞춘다**: 경로마다 같은 색의 화살촉 마커(`<marker>`)를 따로 정의해, 어느 패키지·경로의 흐름인지 선 색만으로 구분되게 한다.
- 색 값은 §2 토큰 팔레트에서 가져와 §6 밝은 배경 위에서 라이트·다크 양쪽 대비를 확인한다.
- **한 다이어그램에선 한 색 체계만** 쓴다 — 흐름 역할색(§2)과 패키지색(§7.1)을 섞지 않는다.

> [!warning] 색만으로 의미를 떠넘기지 않는다 (§2와 동일)
> 패키지 구분도 색 + 라벨(패키지명·경로 글자) + 범례를 함께 쓴다. 색약·흑백 인쇄에서도 라벨로 구분되게 한다.

### 7.2 라이프사이클(호출 시점) 흐름도 — 고유 장치

"호출 시점(시간축) 라이프사이클"을 그릴 때 쓰는 약속:

- **세로 시간축**: 위에서 아래로 호출 순서. 사용자 입력(상단) → 처리 → 모델·결과(하단).
- **경로 레인 A·B·C**: 진입 경로가 여러 개면 각 경로에 글자(A/B/C)를 붙이고, 곁가지는 좌/우로 빼고, 범례에서 한 줄로 정의한다. 경로별 화살표 색은 §7.1을 따른다.
- **수렴점 마커 `◆`**: 여러 경로가 같은 코드(같은 setter·라우터)로 만나는 지점을 마름모 글리프 + 굵은 테두리(2.5px)로 강조한다("◆ 양 경로 공통 수렴점").
- **비동기 경계 마커 `☁`**: OT 채널·메시지 큐·네트워크 등 비동기 경계를 구름 글리프로 표시하고, 발생(전송 전)과 재생(수신 후)을 이 경계로 나눈다.
- **사이드 박스(점선 경계)**: 본 흐름에 항상 타지는 않는 선택·대조 경로(예: 특정 앱 전용 iframe)는 점선 테두리 박스로 본 골격에서 분리하고, 복귀는 점선 화살표로 그린다.
- **개요도 + 상세도 짝**: 전체 흐름을 한 장으로 주고, 그중 복잡한 박스(예: "모델 반영")는 별도 상세 SVG로 확대해 해당 섹션에 임베드한다.
- 제목 아래 **한 줄 부제**와, ◆ 수렴 · ☁ 비동기 · ▶ 시작 · ── 점선=경계/복귀 를 설명하는 **마커 범례**를 둔다.

## 8. 파일·임베드 연계

저장 위치, 파일명, `viewBox` 지정, 임베드 문법, 무의존 원칙, 드리프트 갱신은 **`conventions.md` §7**을 따른다(여기서 중복하지 않는다). 이 파일은 **시각 품질**만 규정한다.

## 9. 작성 전/후 체크리스트

SVG를 만들거나 갱신하기 전후로 아래를 확인한다.

- [ ] 색 체계가 한 가지인가 — 흐름 역할색(§2) **또는** 패키지색(§7.1) 중 하나만 썼는가(혼용 금지)?
- [ ] (여러 패키지·계층이 얽힌 흐름이면) 패키지별로 색을 구분하고, 같은 패키지는 같은 색이며, **범례에 색=패키지 매핑을 명시**했는가?
- [ ] 색만으로 의미를 구분하지 않고 모양·라벨을 병행했는가?
- [ ] 시작 터미널 1개에서 출발하고, 정상 끝 / 오류 끝이 색으로 구분되는가?
- [ ] 노드가 §3의 표준 모양을 따르는가(처리=사각형, 판단=마름모, 저장소=원통 등)?
- [ ] (라이프사이클·호출 시점 흐름도면) 시간축 방향·경로 레인·수렴점/비동기 경계 마커·마커 범례를 갖췄는가(§7.2)?
- [ ] 픽토그램이 인라인이고 단순하며 라벨을 보조하는가?
- [ ] 최소 글자 크기(13~14px+)와 텍스트 대비를 지켰는가?
- [ ] 명시적 밝은 배경 rect가 있어 다크 테마에서도 보이는가?
- [ ] `viewBox`가 있고 외부 폰트·이미지·스크립트 의존이 없는가? (§8)
- [ ] 흐름 방향이 일관되고 선 교차가 최소인가?
- [ ] 한글 라벨이 깨지거나 노드 밖으로 넘치지 않는가?
