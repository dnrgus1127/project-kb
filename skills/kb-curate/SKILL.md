# kb-curate — 지식베이스 무결성 점검·정리 (점검 위임 · 쓰기 게이트)

## 0. 먼저 규칙을 읽는다 (필수)

`${CLAUDE_PLUGIN_ROOT}/skills/kb-conventions/references/conventions.md`를 Read한다.

## 1. 대상 범위

- 프로젝트명이 주어지면 그 폴더만, 없으면 `<vault>` 전체.

## 2. 점검 (kb-curator 에이전트에 위임)

- `kb-curator` 서브에이전트를 띄워 **점검 리포트**를 받는다. 프롬프트에 대상 범위(프로젝트 폴더 또는 vault 전체)와 "리포트만 반환, 보관함 쓰기 금지, 파일 전문 덤프 금지"를 명시한다.
- 리포트에는 7항목(MOC 동기화·깨진 wikilink·고아 문서·깨진 이미지/고아 SVG·정본 테이블·루트 카탈로그·구조 버전)의 발견 사항과 위치·권장 조치가 포함된다.
- **vault 광역 스캔(Glob/Grep/Read)을 메인에서 직접 하지 않는다** — 스캔은 에이전트가 하고, 메인 컨텍스트엔 리포트만 들어온다.

## 3. 승인 게이트

- 점검 리포트를 사용자에게 **그대로 보여준다**(깨진 링크/고아/중복 정본 목록).
- 자명한 정리(MOC에 누락 문서 링크 추가, 카탈로그 한 줄 동기화)는 제안 후 적용.
- 모호한 것(고아 문서 삭제, 정본 재지정)은 **사람 확인 후**에만. 자동 삭제 금지.

## 4. 반영 (스킬이 쓴다)

- 승인된 항목만 메인에서 수정한다. 대상 파일만 정확히 Read → 부분 Edit(전체 vault 재스캔 금지).

## 5. 마무리

- 정리 후 보관함을 **단일 커밋**하도록 권고.

> 에이전트는 점검(읽기)만, 쓰기는 이 스킬만. `kb-status`와 동일한 **감사→승인→쓰기** 구조로, vault 광역 스캔이 메인 세션 컨텍스트를 점유하지 않게 한다.
