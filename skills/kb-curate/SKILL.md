# kb-curate — 지식베이스 무결성 점검·정리

## 0. 먼저 규칙을 읽는다 (필수)

`${CLAUDE_PLUGIN_ROOT}/skills/kb-conventions/references/conventions.md`를 Read한다.

## 1. 대상 범위

- 프로젝트명이 주어지면 그 폴더만, 없으면 `<vault>` 전체.

## 2. 점검 항목

1. **MOC 문서지도 동기화**: 폴더의 실제 `.md` 문서가 MOC 영역 목록에 모두 링크돼 있는가? 없으면 누락분을 보고/제안.
2. **wikilink 무결성**: 본문의 `[[제목]]`이 실제 존재하는 파일을 가리키는가? 깨진 링크 목록화.
3. **고아 문서(orphan)**: 어디서도 링크되지 않는 문서 탐지(MOC 제외).
4. **이미지 임베드**: `![[*.svg]]` / `![](images/...)` 임베드가 `images/`의 실제 파일을 가리키는가? 깨진 임베드 + 어디서도 참조되지 않는 **고아 SVG** 보고.
5. **정본 테이블 정합성**: MOC 정본 테이블의 정본·관련 문서 링크가 유효한가? 같은 주제에 정본이 둘 이상인지(중복 정본) 점검.
6. **루트 카탈로그 동기화**: `<vault>/📚 프로젝트 카탈로그.md`에 모든 활성 프로젝트가 한 줄씩 있고, 링크가 유효하며, 상태가 그럴듯한가?
7. **구조 버전**: 영역 4폴더·images/·MOC가 갖춰져 있는가? 누락 시 마이그레이션 제안(구버전 프로젝트 대비).

## 3. 검색 방법 (한글 경로 안전)

- `Glob`/`Grep`으로 파일·링크를 수집한다. 셸을 쓸 때는 경로를 따옴표로 감싼다.
- 링크 추출은 `[[...]]`, `![[...]]`, `](images/...)` 패턴.

## 4. 보고 → 정리

- 먼저 **점검 결과 리포트**를 사람에게 보여준다(깨진 링크/고아/중복 정본 목록).
- 자명한 정리(MOC에 누락 문서 링크 추가, 카탈로그 한 줄 동기화)는 제안 후 적용.
- 모호한 것(고아 문서 삭제, 정본 재지정)은 **사람 확인 후**에만. 자동 삭제 금지.

## 5. 마무리

- 정리 후 보관함을 **단일 커밋**하도록 권고.
