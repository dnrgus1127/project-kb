# kb-status — 코드↔문서 드리프트 감사·갱신 (승인 게이트)

## 0. 먼저 규칙을 읽는다 (필수)

`${CLAUDE_PLUGIN_ROOT}/skills/kb-conventions/references/conventions.md`를 Read한다.

작업 대상 **프로젝트가 식별되면** `<vault>/<프로젝트>/conventions.md`(프로젝트별 오버라이드, §12)가 있는지 확인하고, 있으면 Read해 전역 위에 **델타로 병합한 유효 규칙**을 기준으로 카드 이동·드리프트 감사를 수행한다(§0·§10은 잠금 — 전역 우선). 구조/폴더 판단은 이 유효 규칙을 따른다.

## 1. 대상 파악

- **프로젝트**와 **코드 리포 경로**를 확인(인자/대화/ MOC의 코드 리포 경로). 모르면 묻는다.
- **모드 분기**: 요청이 특정 카드의 **상태 이동**(구현 시작 / 구현 완료 / 리오픈)이면 아래 §1-1로 간다. 그 외 "구현 현황 동기화/점검"이면 §2(드리프트 감사)로 간다.

## 1-1. 칸반 카드 이동 (구현 시작 / 완료 / 리오픈)

`conventions.md` §11을 따른다. 대상 카드는 `2. 개발·구현/칸반보드/` 하위에서 제목으로 찾는다(현재 컬럼 폴더를 확인).

| 동작 | 이동 |
|---|---|
| 구현 시작 | `1. 구현 전` → `2. 구현 중` |
| 구현 완료 | `2. 구현 중` → `3. 구현 완료` |
| 리오픈 | `3. 구현 완료` → `2. 구현 중` |

**이동 시 항상 (3가지 함께):**
1. 카드 파일을 대상 컬럼 폴더로 **물리 이동**(`git mv` 권장; 한글·공백 경로는 따옴표).
2. 카드 frontmatter `상태`를 새 컬럼에 맞춰 갱신(`구현 전`/`구현 중`/`구현 완료`). `date`도 오늘로.
3. `칸반보드.md`에서 해당 카드 `[[링크]]`를 이전 컬럼 섹션에서 빼고 새 컬럼 섹션에 추가(부분 갱신). 완료 컬럼은 `- [x]`, 그 외는 `- [ ]`.

**구현 완료일 때만 추가로 (승인 게이트 §3 경유):**
- `구현 현황.md`(정본)의 완료 항목/다음 단계와 `구현 계획.md` Phase 상태에 반영을 **제안**하고, 승인된 항목만 §4로 갱신한다.
- MOC `[!success]` 진행 현황 블록 갱신을 제안한다.

> 단순 이동(시작/리오픈)은 위 3가지만으로 끝난다. 정본/MOC 동기화 제안은 **완료**에서만 한다. 끝에 §5(정합성·마무리)로 간다.

## 2. 감사 (kb-status-pm 에이전트에 위임)

- `kb-status-pm` 서브에이전트를 띄워 **드리프트 리포트**를 받는다. 프롬프트에 프로젝트 폴더와 코드 리포 경로, "근거(커밋/파일) 기반, 보관함 쓰기 금지"를 명시한다.
- 리포트에는 일치/드리프트 항목, 흐름도 갱신 필요 여부, Phase 제안이 포함된다.

## 3. 승인 게이트 (중요)

- 드리프트 리포트를 사용자에게 **그대로 보여주고**, 어느 항목을 반영할지 고르게 한다(AskUserQuestion 또는 목록 확인).
- **승인되지 않은 항목은 반영하지 않는다.** 추측 기반 자동 덮어쓰기 금지.

## 4. 반영 (스킬이 쓴다)

- 승인된 항목만 [[구현 현황]]·[[구현 계획]]의 Phase·관련 문서에 **부분 갱신**(Read → 섹션 Edit). frontmatter `date` 갱신.
- 흐름도 갱신이 승인되면 `<프로젝트>/images/`의 SVG를 다시 그리고 본문 임베드를 확인. SVG는 `conventions.md` §7과 `references/diagram-design.md`(시각 품질·패키지별 색 구분·§9 체크리스트)를 따른다.
- MOC의 `[!success]` 진행 현황 블록을 갱신.

## 5. 정합성 + 마무리

- `kb-curate`를 호출해 링크·이미지·정본·카탈로그를 점검한다.
- 무엇을 근거로 무엇을 갱신했는지 요약 보고. 보관함 단일 커밋 권고.

> 에이전트는 읽기/감사만, 쓰기는 이 스킬만. 코드와 충돌 시 코드가 기준이지만, **문서 반영은 항상 사람 승인을 거친다.**
