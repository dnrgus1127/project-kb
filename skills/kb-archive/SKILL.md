# kb-archive — 프로젝트 아카이브 처리

## 0. 먼저 규칙을 읽는다 (필수)

`${CLAUDE_PLUGIN_ROOT}/skills/kb-conventions/references/conventions.md`를 Read한다.

## 1. 대상·사유 확인

- 아카이브할 프로젝트명과 사유(완료/중단/보류)를 확인한다. 되돌리기 어려운 이동이므로 **사용자 확인 후** 진행.

## 2. 이동

- `<vault>/_archive/`가 없으면 만든다(확인 후).
- `<vault>/<프로젝트>/` 전체를 `<vault>/_archive/<프로젝트>/`로 이동한다.
- 한글·이모지·공백 경로는 따옴표 처리. 가능하면 `git mv`로 이력을 보존(보관함은 로컬 git).

## 3. 카탈로그·링크 갱신

- 루트 카탈로그 `<vault>/📚 프로젝트 카탈로그.md`에서 해당 프로젝트의 **상태를 📦 아카이브**로 바꾸고, 링크 경로를 `_archive/` 기준으로 갱신(또는 비활성 표시).
- 아카이브된 프로젝트를 가리키던 다른 문서의 `[[링크]]`가 깨지지 않는지 확인.

## 4. 정합성 + 마무리

- `kb-curate`를 호출해 깨진 링크/카탈로그 정합성을 점검한다.
- 무엇을 어디로 옮겼는지 요약 보고. 보관함 단일 커밋 권고.

> 아카이브는 삭제가 아니다. 내용은 `_archive/`에 보존된다. 영구 삭제는 사용자가 명시적으로 요청할 때만.
