# kb-update — 지식베이스 갱신 (라우팅 + 정본 + 시각자료)

## 0. 먼저 규칙을 읽는다 (필수)

`${CLAUDE_PLUGIN_ROOT}/skills/kb-conventions/references/conventions.md`를 Read한다.

## 1. 대상 파악

- **프로젝트**: 인자/대화에서 식별. 모르면 `<vault>` 하위 폴더 목록을 보여주고 고르게 한다.
- 프로젝트 폴더가 없으면 `kb-init`을 먼저 안내(여기서 새 구조를 만들지 않는다).

## 2. 라우팅 (어디에 넣을지)

`conventions.md`의 영역 매핑으로 내용을 분류한다:
- 개요/설계 → `1. 개요·전략`
- 계획/아이디어/현황/흐름 → `2. 개발·구현`
- 실행/운영 → `3. 운영·셋업`
- 사용자 매뉴얼 → `4. 사용설명서`

적합한 기존 문서가 있으면 그 문서의 **해당 섹션**에 넣는다. 없으면 새 문서를 어디에 만들지 사용자에게 제안·확인한다.

## 3. 정본(canonical) 검사

- 이 주제의 정본이 MOC 정본 테이블에 이미 있는지 확인.
- 있으면 **중복 정본을 새로 만들지 않는다.** 기존 정본을 갱신하거나, 새 문서가 정본을 `[[링크]]`로 가리키게 한다.
- 새로운 정본이 필요하면 MOC 정본 테이블에 한 줄 추가.

## 4. 쓰기 (부분 갱신)

- 대상 문서를 **Read → 해당 섹션만 Edit/갱신**한다. 통째 덮어쓰기 금지.
- frontmatter의 `date`를 오늘로 갱신.
- 새 문서를 만들면 적절한 스텁/형식과 frontmatter(tags 포함)를 갖춘다.
- 관련 문서에 `[[wikilink]]`를 건다.

## 5. 시각 자료 (필요 시)

- 흐름/구조/시퀀스를 추가·변경하면 **SVG를 `<프로젝트>/images/`에 생성**하고 본문에 임베드(`![[파일.svg]]`).
- 텍스트로 충분하면 Mermaid 코드펜스로 본문에 둔다.
- SVG 작성은 `conventions.md` §7(저장·임베드)과 `references/diagram-design.md`(색 토큰·노드 종류별 모양·종단점 강조·픽토그램·가독성·테마 안전)를 따른다. 그리기 전후로 `diagram-design.md` §8 체크리스트를 점검한다.

## 6. MOC 갱신 + 정합성

- 새 문서를 만들었으면 MOC 영역 목록에 `[[링크]]`를 추가.
- 진행 현황이 바뀌면 MOC `[!success]` 블록 갱신.
- 끝에 **`kb-curate`를 호출**해 링크·이미지·정본·카탈로그 정합성을 점검한다.

## 7. 마무리

- 무엇을 어디에 어떻게 반영했는지 요약 보고.
- 보관함 단일 커밋 권고.

> **에이전트 연계**: 기획 초안이 필요하면 `kb-planner`, 리스크 검토가 필요하면 `kb-risk-reviewer`를 먼저 돌려 결과를 받고, **기록은 이 스킬이** 한다(에이전트는 보관함에 쓰지 않는다).
