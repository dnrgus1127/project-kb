# kb — 지식베이스 검색·회수 (읽기 전용)

이 스킬은 `kb-*` 쓰기 스킬들의 **읽기 짝**이다. 주어진 질문/작업과 관련된 문서를 보관함에서 찾아 핵심을 회수하고, 그 지식 위에서 작업을 이어가게 한다.

## 0. 먼저 규칙을 읽는다 (필수)

`${CLAUDE_PLUGIN_ROOT}/skills/kb-conventions/references/conventions.md`를 Read한다. 보관함 taxonomy(§1)와 vault root 해석(§0)을 그대로 따른다.

## 1. 읽기 전용 원칙

- 이 스킬은 보관함을 **절대 수정하지 않는다.** (Write/Edit 하지 않음)
- 회수한 정보를 기록·반영해야 하면 직접 쓰지 말고 `kb-update`(프로젝트 문서) 또는 `kb-note`(단발 노트)로 안내한다(쓰기 단일 경로 — `conventions.md` §10).

## 2. vault root 해석 + 질의 확보

- `conventions.md` §0으로 vaultRoot를 해석한다(config 파일 → 환경변수 `KB_VAULT_ROOT` → 없으면 사용자에게 묻기). **검색만 하므로 폴더를 새로 만들지 않는다.**
- 인자/대화에서 **무엇을 찾을지**(질의: 키워드·질문·진행할 작업)를 확보한다. 비어 있으면 무엇을 찾을지 되묻는다.

## 3. 범위 결정 (현재 프로젝트 우선)

- cwd·레포명에서 후보 프로젝트명을 뽑아 `<vault>/` 하위 폴더 및 루트 카탈로그(`📚 ...`) 항목과 매칭한다(대소문자·부분일치 허용).
- 매칭되면 그 프로젝트를 **우선 범위**로 둔다. 매칭이 없거나 질의가 여러 프로젝트를 가로지르면 처음부터 **전체 vault**를 대상으로 한다.
- 우선 범위에서 충분한 근거를 못 찾으면 자동으로 **전체 vault로 확장**한다.

## 4. 검색은 Explore 서브에이전트로 격리

검색·독해로 메인 컨텍스트가 파일 덤프로 오염되지 않도록 **Explore 서브에이전트**에 위임한다. 서브에이전트는 cwd·현재 대화를 상속하지 않으므로 프롬프트에 다음을 **명시**해 넘긴다:

- 해석된 vaultRoot 절대경로
- 우선 프로젝트(있으면)와 "근거가 부족하면 전체 vault로 확장하라"는 지시
- 질의(키워드/질문/작업 설명)

서브에이전트에 줄 검색 지침:

1. **구조 우선**: 루트 카탈로그(`📚 ...`)로 프로젝트 지도를 잡고 → 대상 프로젝트 MOC(`📑 ...`)의 "어디부터 읽나" 진입점·정본 테이블을 확인한 뒤 → 4영역 문서로 좁힌다.
2. **광역 폴백**: 질의 키워드로 vault 전체를 grep해, 구조 밖 문서(평면 노트·기타 폴더 포함)도 폴더명을 가정하지 않고 포착한다.
3. 파일 **전문을 덤프하지 않는다.** 반환은 세 가지: (a) 질의에 답하는 핵심 요약, (b) 출처 파일 경로 랭킹 목록 + 각 한 줄 관련 사유, (c) 관련 MOC/정본 포인터.

> 서브에이전트를 쓸 수 없는 환경이면 메인에서 직접 (읽기 전용으로) 같은 절차로 검색한다 — 구조 우선 → 광역 grep 폴백.

## 5. 회수 결과 활용

- 서브에이전트의 요약·경로를 받아, 필요한 **원본만** 추가로 Read해 디테일을 확인한다(전부 다시 읽지 않는다).
- 그 지식 위에서 사용자의 실제 작업/질문을 이어간다. 회수한 내용을 쓸 때는 **출처 문서 경로를 밝힌다.**
- 문서가 코드 동작과 충돌하면 `conventions.md` §6에 따라 **코드 리포가 기준**임을 전제한다.

## 6. 결과 없음 처리

- 관련 문서를 못 찾으면 솔직히 없다고 보고한다(없는 내용을 지어내지 않는다).
- 아직 KB에 기록되지 않았을 수 있음을 알리고, 기록이 필요하면 `kb-update`/`kb-note`를 제안한다.

> 이 스킬은 **읽기만** 한다. 검색 결과의 기록·반영은 항상 쓰기 스킬(`kb-update`/`kb-note`)이 수행한다.
