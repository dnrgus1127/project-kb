# 설계: 프로젝트별 conventions 오버라이드

- 날짜: 2026-06-25
- 대상: project-kb 플러그인
- 상태: 승인됨 (구현 진행)

## 배경 / 문제

`skills/kb-conventions/references/conventions.md`가 **모든 프로젝트 공통 규칙의 단일 출처**이며, 프로젝트별로 폴더 구조나 규칙을 다르게 가져갈 방법이 없다. 일부 프로젝트는 전역 taxonomy(`00·1~5·90·91·92` + `images/` + MOC)와 다른 구조나 명명 규칙이 필요할 수 있다.

## 목표

특정 프로젝트가 **전역 규칙을 베이스로 두되, 그 프로젝트만의 폴더 구조·규칙을 델타로 오버라이드**할 수 있게 한다.

## 확정된 결정

| 항목 | 결정 |
|---|---|
| 위치 | `<vault>/<프로젝트>/conventions.md` (프로젝트 폴더 루트) |
| 파일명 | `conventions.md` — 전역과 동일(위치가 달라 충돌 없음, "같은 종류의 규칙 문서"임을 드러냄) |
| 의미론 | **델타 오버라이드** — 전역이 베이스, 프로젝트 파일이 충돌 항목에서 우선. 항목/섹션 단위 병합(전체 치환 아님). 어떤 섹션이든 가능(잠금 섹션 제외) |
| 잠금 섹션 | §0(보관함 루트 해석), §10(쓰기·커밋 원칙)은 오버라이드 불가 — 프로젝트 파일이 건드리면 무시하고 전역을 따른다 |
| 생성 시점 | **온디맨드** — kb-init은 기본은 전역대로 스캐폴딩, 커스텀이 필요할 때만 작성. 기존 프로젝트엔 사용자가 직접 추가 가능 |
| 메타 취급 | 지식 문서가 아닌 메타/설정 파일. MOC 영역 목록·정본 테이블에 올리지 않으며, 정합성 점검에서 고아/지식문서로 오탐하지 않는다 |

## 발견·병합 메커니즘 (선택된 접근법 A)

오버라이드 *작동 방식*의 정의는 전역 `conventions.md` **한 곳**(신설 §12)에만 둔다. 각 kb-* 스킬은 기존처럼 전역 conventions.md를 Read하고, "0. 규칙 읽기" 단계에 **한 줄**을 추가해 — 프로젝트가 식별되면 `<vault>/<프로젝트>/conventions.md`도 Read하여 전역 위에 델타로 병합한 **유효 규칙(effective rules)**으로 작업한다.

- 기각한 대안 B(각 스킬이 오버라이드 로직 개별 문서화 → 중복·드리프트), C(전용 resolver 문서 → 프로즈 기반 LLM 스킬엔 과설계).

## 변경 범위

### 1. 규칙 단일 출처
- `skills/kb-conventions/references/conventions.md`: **§12 "프로젝트별 오버라이드"** 신설(병합 규칙·잠금 섹션·메타 취급·적용·생성). §1에 §12로의 포인터 한 줄.
- `skills/kb-conventions/SKILL.md`: 템플릿 목록에 `project-conventions.md` 추가, 오버라이드 메커니즘이 §12에 있음을 명시.

### 2. 템플릿 신설
- `skills/kb-conventions/references/templates/project-conventions.md`: 헤더 고정 문구("전역을 베이스로 한 델타, 안 적은 항목은 전역대로") + 작성 가이드 주석(§1 폴더 taxonomy 교체 예) + placeholder(`{{PROJECT_NAME}}`/`{{PROJECT_SLUG}}`/`{{DATE}}`).

### 3. 스킬 변경
- **kb-init**: §1에 "커스텀 구조 여부" 질문(온디맨드) 추가. §3에 — 커스텀이면 템플릿으로 프로젝트 `conventions.md`를 먼저 만들고 **유효 규칙대로** 스캐폴딩, 아니면 전역대로(파일 안 만듦).
- **kb-update / kb-status / kb(검색)**: "0. 규칙 읽기"에 병합 한 줄 추가(라우팅·감사·검색 범위가 유효 규칙 기준).
- **kb-curate**: 병합 한 줄 + 점검 위임 시 커리에이터에게 "프로젝트 오버라이드는 메타 파일, 유효 구조 기준 점검" 전달.
- **kb-note / kb-archive / kb-set-vault**: 변경 없음(kb-note는 비프로젝트 노트라 비대상, kb-archive는 폴더 통째 이동이라 파일이 함께 따라감).

### 4. 에이전트 변경
- **kb-curator**: 시작 전 — 프로젝트 `conventions.md`가 있으면 병합한 유효 구조를 점검 기준으로. MOC 동기화(1)·고아(3) 점검에서 프로젝트 `conventions.md`를 메타로 제외. 구조 버전(7) 점검을 유효 구조 기준으로.
- **kb-status-pm**: 시작 전 병합 한 줄(유효 구조 기준 감사).

### 5. 버전
- `plugin.json`: `0.4.4` → `0.4.5`.

## 엣지 케이스
- 프로젝트 파일이 잠금 섹션(§0/§10)을 건드리면 무시하고 전역을 따른다.
- 빈 델타(아무 섹션도 안 적음)면 전역과 동일하게 동작한다.
- 전역에 새 섹션이 추가돼도 프로젝트 파일은 "없는 건 전역 따름"이라 자동 호환된다.
